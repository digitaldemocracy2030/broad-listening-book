## ブロードリスニングとしてみたときのパブリックコメント

ブロードリスニングに関する取り組みをしていると、「行政は既にパブリックコメントという形でブロードリスニングでやってるじゃないか」という意見を貰うことがよくあります。しかし、パブリックコメントをブロードリスニングとして見たときに、パブリックコメントには多くの問題点があります。

なお、本コラムでは国（省庁）が実施するパブリックコメントを対象に分析します。自治体が実施するパブリックコメントは、根拠となる条例や対象（計画案、条例案など）が異なりますが、「専門的な文書に対する意見募集」という構造的な課題は共通しています。

### そもそもパブリックコメントは何を対象に実施されるのか？

そもそもパブリックコメントの根拠はどこにあるのでしょうか？それは行政手続法の第六章にあります。その冒頭の行政手続法第38条を引用します。

```
行政手続法 第六章　意見公募手続等
第三十八条 命令等を定める場合の一般原則
命令等制定機関は、命令等を定めようとする場合には、当該命令等の案及びこれに関連する資料をあらかじめ公示し、意見の提出先及び意見の提出のための期間を定めて広く一般の意見を求めなければならない。
https://laws.e-gov.go.jp/law/405AC0000000088?utm_source=chatgpt.com#Mp-Ch_6
```

この法律をざっくりと要約すると次のようになります。

- 「命令等（政令・省令・審査基準・処分基準・行政指導指針など）」を定めるときに (第38条)
- 事前に案を公示して (第39条1項・2項)
- 一般から意見を募り（第39条1項・3項）
- その意見を考慮し（第42条）
- 結果を公示する（第43条・第45条）

つまり、パブリックコメントは、あらかた出来上がった命令の最終チェックに使われるということです。そして意見を出しているのは大半が、その命令によって直接的に影響を受ける企業と、ごく一部の殊勝な個人です。そのため、ほとんどのパブリックコメントは10件以下の意見しか集まっていません。

### ダブルダイヤモンドによるパブリックコメントとブロードリスニングの比較

ブロードリスニングとパブリックコメントを比較検討するために、まずはデザイン思考におけるダイヤモンドモデルという考え方を導入します。

![ダブルダイヤモンドモデル](https://www.designcouncil.org.uk/fileadmin/_processed_/4/9/csm_Framework_for_Innovation_transparent_14a10de530.png)

ダブルダイヤモンドモデルでは、広い指向で物事を考えるための発散のフェーズと、深く集中して考えるための収束のフェーズがあります。そして発散と収束を二回繰り返すことで、「課題発見・課題定義」と「解決策の検討・解決策の実施」という二つのダイヤモンドを作り出し、「正しい問題を見つける」「正しい解決を見つける」という二段階を経て、時には後戻りをしてフィードバックしていくことで問題を解決していきます。

では、これに対してパブリックコメントはどこを担っているかというと、実施段階における微修正に留まっているのが実情です。つまりダブルダイヤモンドにおける最後の収束段階なのです。なぜなら、行政官がある程度固めて、これで世に出せるという段階になった命令に対しての修正案を募集するからです。

現在、ブロードリスニング技術の研究開発は、ダブルダイヤモンドの後半部分、つまり「解決策の検討・実施」フェーズにいかにリーチするかという課題に取り組み始めています。例えば、大規模熟議による合意形成支援などがその一例です。しかし、現行のパブリックコメント制度をそのままブロードリスニングの枠組みに当てはめることには、根本的な限界があります。

そもそもパブリックコメントは、行政が作成した案の「チェック」と、手続き的な正統性の担保のための仕組みです。「広く社会の声を吸い上げる」という目的で設計されたものではありません。

そのため、幅広い人にわかりやすく伝える工夫よりも、「形式的な要件を満たすこと」が優先されがちです。また、「議題の設定」や「論点の整理」も行政側が一方的に行うため、そもそも何を問い、何を聴かないかは最初から決められています。

本来ブロードリスニングと呼べるためには、議題設定の段階から市民が関与できること、ルール案が固まる前の段階でも参加できること、といった要素が必要です。しかし現在のパブリックコメントは「最後に一応聞きます」という性格が強く、ブロードリスニングとは本質的に異なる仕組みなのです。

またFacebookやXなどのSNSでは、草の根的にパブコメへの意見投稿を促すポストが流れていました。例えば次のようなハッシュタグをつけ、パブリックコメントへの意見投稿を促すポストが多く見られました。

こういった不安を煽る傾向が強い文言は、不安傾向の強い人を感化しやすく、結果として大量の意見が集まることになりました。

一方でパブリックコメント制度は仕組み上、同一内容の意見が多数提出された場合であっても、その数が考慮の対象となる制度ではありません。

```
パブリック・コメント制度では、提出された意見の「量」ではなく「内容」を考慮します。同一内容の意見が多数提出された場合であっても、その数が考慮の対象となる制度ではありません。
https://public-comment.e-gov.go.jp/contents/about-public-comment/
```

では意見の量を考慮しないとされているにもかかわらず、なぜNPO団体は大量の意見を送ろうとするのでしょうか？これには３つの理由が考えられます。

第一に、大量の意見が集まることで、マスメディアなどで取り上げられやすくなり、世論喚起の効果があるためです。大量の意見が集まることで、行政側も無視できないという圧力をかけることができます。

第二に、パブリックコメントは行政官が全件目を通す義務があり、また似たようなコメントを手作業で集約する必要があるため、大量の意見が集まると、DDoS攻撃のように行政の業務を妨害する効果を持ってしまいます。 一部のNPO団体はこのような業務妨害効果を狙って、大量の意見投稿を呼び掛けていたと考えられます。 さらに敷衍して考えると「嫌がらせを止める能力を持つことで、影響力を引き上げる」といった戦略的な意図も考えられます。

第三に、パブリックコメント制度は、提出された意見の「量」は無視されることが明言されているため、大量の意見を送ることで「大量の市民の意見を無視した政府」という印象操作を行うことができるためです。

こういった背景があることから、今後もパブリックコメントは大規模動員された意見募集が継続する可能性が高く、さらには本人確認も不要であるため、ボットなどによる自動投稿も横行する可能性があります。他国から攻撃されるサーフェイスの一つになり得ます。

今回のケースでは人間が手作業でコピペを行って投稿していたため、大量の重複意見が発生しましたが、将来的にはAI技術を活用して自動的に意見を生成し、投稿することも可能になるでしょう。これにより、単純な重複排除では対処できない、より巧妙な大量意見投稿が横行する可能性があります。

なお、DD2030では、一部のパブリックコメントの意見に対して、情報開示請求を行い原文を入手し、自然言語処理技術を活用して大量投稿の検出・重複判定を試みました。AIを活用して大量投稿してくるのであれば、こちらもAIを活用して対抗する必要があるためです。

### ブロードリスニングとパブリックコメントの役割分担

ここまで見てきたように、パブリックコメントには構造的な限界があります。しかし、だからといってパブコメを廃止すべきだという話ではありません。

行政手続法第38条には「広く一般の意見を求めなければならない」と明記されています。パブリックコメントには、本来「広く国民の声を聞く」という役割があるのです。

しかし実運用はどうでしょうか。本コラムで見てきたように、9割のパブコメは10件以下の意見しか集まらず、集まる意見の大半は専門家や利害関係者からのものです。「広く一般の意見を求める」という法の趣旨と、実態との間には大きな乖離があります。

結果として、パブコメは事実上「あらかた出来上がった命令案の最終チェック」——専門家や利害関係者が、法的・実務的な観点から問題点を指摘する場——として機能しています。この機能自体は重要ですが、「広く聞く」という本来の役割は果たせていません。

ブロードリスニングは、この「広く聞く」部分を担うことができます。ダブルダイヤモンドで言えば、前半の「課題発見・課題定義」をブロードリスニングが担い、後半の「解決策の実施」段階でパブコメが最終チェックを行う——このような役割分担が可能になったのです。

理想的なフローは以下のようになります。

1. **ブロードリスニング**：広く市民の声を集め、論点を発見する
2. **政策立案**：発見した論点を踏まえて政策案を作成する
3. **パブリックコメント**：専門家・利害関係者が最終チェックを行う
4. **ブロードリスニング**：パブコメで集まった意見をAIで構造化し、対応を検討する

このように、ブロードリスニングとパブリックコメントは対立するものではなく、民主主義プロセスの異なるフェーズを担う補完的な関係にあるのです。
